# Database Schema & Vector Search

LokKatha AI relies on Supabase (PostgreSQL) extended with `pgvector` to enable blazing fast semantic retrieval of cultural folklore.

## 1. Extension Requirements

Before initializing the database, ensure `pgvector` is enabled in your Supabase project:
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

## 2. Main `stories` Table

The primary table stores both the literal content of the folktale and the high-dimensional vector embeddings generated by Google Gemini.

```sql
CREATE TABLE public.stories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title TEXT NOT NULL,
    summary TEXT,
    content TEXT NOT NULL,
    metadata JSONB,
    embedding VECTOR(768),
    search_document TSVECTOR GENERATED ALWAYS AS (
        setweight(to_tsvector('english', coalesce(title, '')), 'A') ||
        setweight(to_tsvector('english', coalesce(summary, '')), 'B') ||
        setweight(to_tsvector('english', coalesce(content, '')), 'C')
    ) STORED,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

### JSONB `metadata` Structure
The `metadata` column allows flexible filtering by Region, Era, and Language.

## 3. Retrieval Function (HNSW / IVFFlat)

To retrieve stories, we use a PostgreSQL function that calculates the Cosine Distance (`<=>`) between the user's query vector and the story database.

```sql
CREATE OR REPLACE FUNCTION match_stories (
  query_embedding VECTOR(768),
  match_threshold FLOAT,
  match_count INT
)
RETURNS TABLE (
  id UUID,
  title TEXT,
  summary TEXT,
  content TEXT,
  metadata JSONB,
  similarity FLOAT
)
LANGUAGE sql STABLE
AS $$
  SELECT
    id,
    title,
    summary,
    content,
    metadata,
    1 - (stories.embedding <=> query_embedding) AS similarity
  FROM stories
  WHERE 1 - (stories.embedding <=> query_embedding) > match_threshold
  ORDER BY similarity DESC
  LIMIT match_count;
$$;
```

## 4. Performance Indexes

To ensure sub-50ms retrieval on large datasets, we apply an HNSW index on the embedding column:

```sql
CREATE INDEX ON stories USING hnsw (embedding vector_cosine_ops)
WITH (m = 16, ef_construction = 64);
```
